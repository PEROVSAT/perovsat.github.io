# Permanent Magnet Module

## Overview

The **Permanent Magnet** module models the torque generated by a permanent magnetic dipole interacting with Earth's magnetic field. It serves as the primary restoring element of the Passive Magnetic Attitude Control (PMAC) system.

The module is implemented as a custom **Basilisk DynamicEffector**, allowing the magnetic torque to be applied directly to the spacecraft dynamics during numerical integration.

Unlike an active magnetic control system, the permanent magnet produces torque passively without requiring sensors, actuators, or control algorithms.

---

## Purpose

The Permanent Magnet module performs the following tasks:

- Reads the Earth's magnetic field from the World Magnetic Model (WMM).
- Converts the magnetic field into the spacecraft body frame.
- Computes the magnetic torque generated by the permanent magnet.
- Applies the resulting torque to the spacecraft dynamics.

---

## Physical Principle

A magnetic dipole placed within an external magnetic field experiences a restoring torque given by

$$
\mathbf{T}
=
\mathbf{m}
\times
\mathbf{B}
$$

where

- $\mathbf{T}$ is the magnetic torque,
- $\mathbf{m}$ is the permanent magnet dipole moment,
- $\mathbf{B}$ is the Earth's magnetic field.

The torque naturally rotates the spacecraft until the permanent magnet aligns with the surrounding magnetic field.

Because no electrical power is required, permanent magnets provide a lightweight and highly reliable method of passive attitude stabilization.

---

## Basilisk Implementation

The Permanent Magnet module inherits from both

```cpp
SysModel
```

and

```cpp
DynamicEffector
```

This design allows the module to participate in the Basilisk simulation loop while contributing external torques directly to the spacecraft equations of motion.

The primary configurable parameter is

```cpp
Eigen::Vector3d magDipole_B;
```

which defines the magnetic dipole moment of the permanent magnet in the spacecraft body frame.

The module also subscribes to the magnetic field message

```cpp
ReadFunctor<MagneticFieldMsgPayload> magFieldInMsg;
```

and publishes the computed body-frame torque through

```cpp
Message<CmdTorqueBodyMsgPayload> cmdTorqueOutMsg;
```

for logging and visualization.

## Outputs

The module computes and publishes

$$
\mathbf{T}
=
\begin{bmatrix}
T_x\\
T_y\\
T_z
\end{bmatrix}
\quad [\mathrm{N\,m}]
$$

which represents the magnetic torque applied to the spacecraft body.

---

## Implementation Workflow

During each simulation step, the Permanent Magnet module executes the following sequence:

1. Read the latest magnetic field from the WMM module.
2. Obtain the current spacecraft attitude from the dynamics integrator.
3. Convert the spacecraft attitude from Modified Rodrigues Parameters (MRPs) to a Direction Cosine Matrix (DCM).
4. Rotate the magnetic field from the inertial frame into the spacecraft body frame.
5. Compute the magnetic torque using the cross product.
6. Apply the computed torque to the spacecraft dynamics.

---

## Coordinate Transformation

The World Magnetic Model provides the magnetic field in the inertial frame

$$
\mathbf{B}_N
$$

The spacecraft attitude is represented using Modified Rodrigues Parameters (MRPs). These are converted into a Direction Cosine Matrix

$$
\mathbf{C}_{BN}
$$

which transforms vectors from the inertial frame to the spacecraft body frame.

The body-frame magnetic field is therefore

$$
\mathbf{B}_B
=
\mathbf{C}_{BN}
\mathbf{B}_N
$$

This transformed field is used in the magnetic torque calculation.

---

## Torque Computation

Once the magnetic field has been transformed into the spacecraft body frame, the restoring torque is computed as

$$
\mathbf{T}
=
\mathbf{m}_B
\times
\mathbf{B}_B
$$

where

- $\mathbf{m}_B$ is the permanent magnet dipole expressed in the body frame.
- $\mathbf{B}_B$ is the magnetic field expressed in the body frame.

The resulting torque is automatically added to the spacecraft dynamics through Basilisk's `DynamicEffector` interface.

## Advantages

The Permanent Magnet module provides several benefits for passive spacecraft stabilization:

- No electrical power consumption.
- No active control algorithms.
- Minimal computational cost.
- Direct integration with Basilisk's dynamics engine.
- Continuous torque computation during numerical integration.

---

## Summary

The Permanent Magnet module models the passive restoring torque generated by a permanent magnetic dipole interacting with Earth's magnetic field. By reading the magnetic field from the World Magnetic Model, transforming it into the spacecraft body frame, and applying the torque through Basilisk's `DynamicEffector` interface, the module enables realistic simulation of passive magnetic attitude stabilization in Low Earth Orbit.
